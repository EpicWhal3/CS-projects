# Corporate Messaging System

Учебный проект на C#, моделирующий корпоративную систему распределения сообщений

## О проекте

В проекте реализована объектная модель системы доставки сообщений между различными типами адресатов.  
Сообщение содержит:

- заголовок;
- тело;
- уровень важности.

Отправка производится через топики, которые передают сообщение адресатам.  
Поддерживаются разные типы адресатов и дополнительные механизмы фильтрации и логгирования.

## Что реализовано

- модель сообщения с заголовком, телом и приоритетом;
- топики, которые отправляют сообщения адресатам;
- адресат-пользователь;
- адресат-мессенджер;
- адресат-дисплей;
- адресат-группа;
- пользователь как конечная точка с хранением полученных сообщений и их статусов;
- возможность отмечать сообщение как прочитанное;
- фильтрация сообщений по важности для конкретного адресата;
- логгирование получения сообщений;
- отдельные реализации для интеграций мессенджера и дисплея;
- драйверы дисплея для разных способов вывода;
- функциональные и unit-тесты с использованием mock-объектов.

## Архитектурные особенности

Проект строится вокруг расширяемой модели доставки сообщений и демонстрирует разделение доменной логики и инфраструктурных деталей.

В реализации отражены:

- структурные паттерны для композиции и расширения поведения адресатов;
- использование прокси/обёрток для фильтрации и логгирования;
- отделение логики доставки от конкретных внешних интеграций;
- изоляция мессенджера и дисплея как независимых компонентов;
- тестируемость логгирования, мессенджера и драйверов вывода.

## Поведение системы

Поддерживаются следующие сценарии:

- сообщение передаётся через топик одному или нескольким адресатам;
- пользователь получает сообщение в статусе “не прочитано”;
- пользователь может отметить сообщение как прочитанное;
- повторная попытка отметить уже прочитанное сообщение обрабатывается как ошибка;
- адресат может игнорировать сообщения ниже заданного уровня важности;
- адресат может быть обёрнут логгированием;
- группа адресатов передаёт сообщение всем вложенным адресатам;
- для дисплея поддерживается очистка перед выводом и работа через драйвер.

## Тестирование

В проекте проверяются следующие сценарии:

- сообщение у пользователя сохраняется как непрочитанное;
- сообщение корректно переводится в статус “прочитано”;
- повторное чтение уже прочитанного сообщения приводит к ошибке;
- фильтр по важности блокирует неподходящее сообщение;
- логгирование срабатывает при получении сообщения;
- сообщение корректно передаётся в мессенджер;
- при нескольких адресатах для одного пользователя сообщение не дублируется ошибочным образом.
