# ATM System

Учебный проект на C#, реализующий систему банкомата с консольным интерфейсом, хранением данных в PostgreSQL и разделением приложения на слои.

## О проекте

В рамках лабораторной разработана система банкомата, поддерживающая два режима работы:

- пользователь;
- администратор.

Для пользователя доступны основные банковские операции:

- создание счёта;
- просмотр баланса;
- снятие средств;
- пополнение счёта;
- просмотр истории операций.

Для администратора предусмотрен отдельный вход по системному паролю.

## Что реализовано

- интерактивный консольный интерфейс;
- выбор режима работы при запуске;
- вход пользователя по номеру счёта и PIN-коду;
- вход администратора по системному паролю;
- операции снятия и пополнения средств;
- просмотр текущего баланса;
- получение истории операций;
- хранение и загрузка данных из PostgreSQL;
- работа с базой данных без ORM;
- разделение приложения на слои/проекты с выделением application-, infrastructure- и presentation-логики;
- тестирование бизнес-сервисов с mock-репозиториями.

## Архитектурные особенности

Реализация оформлена как многослойное приложение с явным разделением ответственности:

- отделение бизнес-логики от консольного интерфейса;
- отделение бизнес-логики от доступа к данным;
- использование репозиториев и сервисов для прикладных сценариев;
- независимость unit-тестов от реальной базы данных;
- возможность тестировать ключевые операции на уровне сервисов.

## Работа с данными

В проекте реализовано:

- персистентное хранение данных в PostgreSQL;
- доступ к данным без ORM;
- ручная работа с инфраструктурным слоем;
- хранение информации о счётах и операциях.

## Бизнес-сценарии

Поддерживаются основные банковские действия:

- создание нового счёта;
- авторизация пользователя;
- пополнение счёта;
- снятие средств;
- проверка достаточности баланса;
- вывод истории операций;
- административный доступ по системному паролю.

При ошибочных операциях приложение должно возвращать сообщения об ошибке и корректно обрабатывать невалидные сценарии.

## Тестирование

В тестах проверяется именно бизнес-логика, а не база данных или консольное представление.

Основные сценарии:

### Снятие средств

- при достаточном балансе счёт сохраняется с обновлённым значением;
- при недостаточном балансе сервис возвращает ошибку.

### Пополнение счёта

- после пополнения счёт сохраняется с корректно обновлённым балансом.

Для тестирования используются mock-репозитории, что позволяет изолировать доменную и прикладную логику от инфраструктуры.
